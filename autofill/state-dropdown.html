State dropdown autofill conflict

These forms test the interaction between DuckDuckGo's autofill identity tooltip and state/province dropdown fields. The autofill tooltip should not appear on custom dropdown inputs that already have their own picker UI.

Form 1: Native <select> for state

Baseline. Autofill already handles native selects correctly — no tooltip shown on the select element.

Form 2: Custom combobox input for state

This uses an <input role="combobox"> with a listbox dropdown. The autofill tooltip should not appear on this field since it already has a dropdown.
