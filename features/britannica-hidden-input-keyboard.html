Britannica hidden-input keyboard repro

Reduced from britannica.com/games/revealed. This keeps only the letter tiles and the off-screen zero-size input that receives focus when a tile is tapped.

Expected behavior: tapping a tile keeps keyboard focus on the puzzle input so you can type letters.

Bug behavior on affected Android WebView builds: the keyboard appears briefly and then dismisses, even though the tile tap called focus() on the hidden input.

Test steps

  1. Tap any tile in the puzzle field.
  2. Try typing a letter.
  3. If the keyboard dismisses, tap Focus visible control input to confirm keyboard behavior on the same page.

Kept: tile button -> hidden input focus handoff, zero-sized off-screen input, single-character entry.

Removed: Next.js app shell, route transitions, networking, storage, hints, analytics, and all puzzle logic beyond single-character entry.
